# 資料治理與保留策略

## 保留規則

1. 站內通知：保留 30 天（超過自動清除）
2. 訂單/票券：保留營運所需歷史資料（不主動刪除）
3. 稽核日誌：建議至少保留 90 天

## 備份策略

1. 每日全量備份（凌晨）
2. 重大部署前強制備份一次
3. 備份檔加密與存放至少 2 個位置

腳本：

- 備份：`scripts/mysql-backup.sh`
- 還原：`scripts/mysql-restore.sh`

## 還原演練

1. 每月至少一次在測試環境演練還原。
2. 驗證：
   - 會員可登入
   - 訂單可查
   - 通知與點數資料一致

## 變更治理

- schema 變更只走 Flyway。
- 舊 `V*.sql` 不可修改。
- 海報/輪播內容調整走 `R__movie_catalog_assets.sql`。
