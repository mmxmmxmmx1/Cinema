# 監控與告警建議

## 1. 即時健康檢查

- 端點：`/api/health`
- 建議每 30-60 秒探測。
- 告警條件：
  - `status != UP`
  - `db != UP`

## 2. 核心營運指標

1. 付款失敗率（24h）
2. 未付款逾時訂單數
3. 取消訂單比例
4. 通知未讀量異常成長
5. 平均與 P95 API 延遲

## 3. 最低可行告警規則

- 連續 3 次 health fail -> 立即告警
- 付款失敗率 > 20% (15 分鐘視窗) -> 告警
- 5xx 比例 > 2% (5 分鐘視窗) -> 告警
- DB 連線失敗 > 0 -> 告警

## 4. 建議工具

- Prometheus + Grafana（監控）
- Loki/ELK（日誌）
- Alertmanager（通知）
